Le CSS

Le langage Css vient completer celui du HTML. Là où ce dernier permet d'afficher, le CSS lui vient définir "comment" l'afficher.
Il faut être vigilant sur la compatibilité des navigateurs avec certaines fonctionnalités récentes de CSS3. Quand un navigateur ne connaît pas une instruction de mise en forme, il l'ignore simplement.
On peut écrire le code CSS à plusieurs endroits différents, le plus conseillé étant de créer un fichier séparé portant l'extension ".css " (exemple :style.css).
En CSS, on sélectionne les portions de la page HTML que l'on souhaite modifier et on change leur présentation avec des propriétés CSS.
sources : openclassrooms.com

FleXbox

FlexBox marque un renouveau de la mise en page en CSS.

En effet, avant l'arrivé de FlexBox, pour déplacer des élèments en Css, il fallait essayer de faire concorder de faire voler les elements dans la page a la maniere d'un télékinésiste ( c'est un peu abusé mais y'a du vrai là dedans...)

Maintenant nous alons pouvoir nous organiser beaucoup plus facilement.

Vocabulaire important pour maitriser FlexBox.

Le conteneur: Flex Container.

Les élèments contenus: Flex Items.

A savoir

A savoir: Les propriétés pour Flex Container sont différentes pour les Flex Items.

Les propriétés pour Flex Container vont s'appliquer sur Flex Items contenur dans ce Flex Container.

Les propriétés pour les Flex Items vont s'appliquer à un ou des Flex Items

Il peut y avoir des structures imbriquées.

Fonctionement.

Tout d'abord il faut appliquer une propriété sur le Flex Container "display: flex" dans le css. (C'est une manière de dire " Ici, on utilise flexbox")

ensuite nous pouvons ajouter les propriétés flex.

  • Flex-direction: Row(par défaut), row-reverse, column, column-reverse(permet de disposer des blocs d'une certaine direction: ('→' '←' '↓' '↑'))
  • Flex-wrap: Nowrap(par défaut),wrap (permet de revenir ou non à la ligne)
  • Flex-flow: Flex-direction, flew-wrap (permet d'ajouter les deux dernieres propriétés.)
  • Justify-content: Flex-start, flex end, center, space-between, space-around, space-evenly
  • Align-items: Flex-start, flex end, center,stretch, baseline
  • Align-content: Flex-start, flex end, center,stretch, space-between, space-around
  • Order: Permet de changer l’ordre d’affichage d’un élément
  • Flex-grow: Permet de changer la taille d’affichage d’un élément en le grossissant
  • Flex-shrink: Permet de changer la taille d’affichage d’un élément en le retrécissant
  • Flex-basis: Permet de definir une taille par défaut
  • Align-self: Permet de changer l'alignement d'un élément en particulier

SASS

Qu'est que SASS?

Par définition c'est un préprocesseur CSS

-Et?

Et ça veut dire qu'il va nous permettre d'écrire du CSS plus simplement et de manière beaucoup plus organisée!

-Pourquoi dire "préprocesseur"?

On appelle cela comme ça car il y une phase de "compilation" pour transformer le SASS en Css !

Il en existe plusieurs comme SASS, les plus connus sont " Less et Stylus. Le plus utilisé reste tout de même SASS avec une utilisation de 66% de part de marché.

SCSS

-Quoi?! Encore un nouveau mot? !

C'est tres simple. Sass a deux syntaxes différentes: sass, et scss (notez l'absence de majuscule).La plus récente est la seconde (scss). Elle est la plus évoluée et la plus utilisée. Son point fort est qu'elle ressemble beaucoup au css.

Sass (avec un grand S!)

C'est quatre points clés:

  1. Des variables en Css
  2. Des syntaxes imbriquées
  3. Des mixins
  4. Des boucles et des conditions

Tout ça permet de rajouter des fonctionnalitées supplémentaire au Css et ça, on adore !

1.Les Variables

Comme dans un langage de programation, nous utiliserons des variables.

Exemple de variable pratique: les couleurs du site. De cette façon si vous souhaitez changer la couleur en cours de route, il vous suffit de changer la valeur stockée dans la variable.

Déclaration de variable: $nom-de-la-variable:valeur;

utilisation: color: $nom-de-la-variable;

Mise en pratique

L'emplacement du fichier sass

Dans le dossier du projet, comme pour: le Css/le js/les images/ video, il est conseillé de créer un fichier "sass" dans lequel se trouvera un fichier "style.scss"

La compilation

Pour que le navigateur puisse lire le fichier, il faut le compiler

Pour cela il faut installer Node Package Manager lien direct pensez à cocher l'option du terminal dans vote IDE ce sera plus simple pour la suite.

and let the compilation begin!

Dans le terminal de votre IDE: "sass source.scss destination.css" (code générique)
sass sass/style.scss css/style.css (formule exemple)

2.La Syntaxe Imbriquée

La syntaxe de Sass va permettre de faire de l'imbrication, comme pour le HTML: avec la notion de parent, enfant et voisin.

Peuvent également être imbriquées: les pseudos-classes(:hover, :focus(par exemple)) ainsi que les média queries(@media)

Les Mixins

En SASS, les Mixins sont à peut près égale aux fonctions des autres languages de programation.

La différence est que les mixins peuvent envoyer plusieurs lignes de Css.

Comment ça fonctionne?

Step 1: déclarer la mixin:

Step2: Appeler la mixin

Déclaration de la Mixin

Pour déclarer une mixin, on commence par écrire "@mixin" suivi du nom que vous souhaitez lui donner

@mixin text-center { text-align:center;}

Comme pour une fonction, il est possible de déterminer les paramètres qui definiront la mixin.

@mixin buttou ($color, $bgcolor){
color:$color;
background-color:$bgcolor;
}
[ici les informations sont des variables)

enfin, il est possible d'ajouter des paramètres par défaut aux mixins

@mixin text-align($align:left){
text-align:$align;
}

Si la variable $align est changée, cela prendre la dessus sur la valeur par défaut.

Appeler la mixin

Pour appeler la mixin, il suffit d'indiquer "@include" suivi du nom de la mixin.

Ex: h1{
//application de la mixin "text-center"
@include text-center;
}

Les boucles

Les boucles permettent de répéter une instruction plusieurs fois par rapport à:

  • un compteur (for)
  • une condition (while)
  • une liste d'éléments (each)
La boucle for

"Pour i allant de 1 à 10 faire..."

La boucle for se fait par le biais d'une variable qui va détenir la valeur du compteur ( i ) suivit de la valeur du début et la valeur de fin.

@for $i from <.début> through <.fin>

@ $i from 1through3 {
.margin-#{$i} {
margin: 1rem*$i;
}
}

La syntaxe #{$i} permet d’utiliser la valeur pourl’afficher dans un sélecteur (dans un chaine demanière générale), vous remarquerez que nous n’avons pas besoin de cette syntaxe pour faire des calculs ou appliquer la valeur sur une propriété par exemple.

La boucle while

"tant que i>10 faire...

La boucle while se fait par l'intermediaire de l'instruction "@while" suivie d'une condition qui va conditionner le nombre d'itération dans la boucle.

@while <.condition>

@while $i <=3{
.margin-#{$i}{
margin: 1rem* $i;
}
$i:$i+1;
}

La boucle each

L'intérêt de la boucle each est de pouvoir parcourir n'importe quelle liste ou collections.

L'instruction @each est suivie d'une variable qui va contenir la valeur d'un element de la liste/collection

@each $item in 1,2,3

les listes peuvent s'écrire de plusieurs façons:

  • 1, 2, 3
  • 1 2 3
  • (1, 2, 3)
  • (1 2 3)

@each $color in red, blue, green {
.bg-#{$color}{
background-color: $color;
}
}

peut aussi s'écrire:

$color in red, blue, green

@each$color in$colors{
.bg-#{$color}{
background-color:$color;
}
}

Et donc avec des collections il suffit de reprendre la logique d'un tableau des autres languages.

Une collection fonctionne avec des associations de clés et de valeurs le tout stocké dans une variable

$grid:(
columns:12
);

Pour récupérer une valeur d’une clé d’une collection vous disposez de la fonction map-get prenant en paramètre la collection et la clé de la valeur souhaitée :

$columns: map-get($grid, "columns");

On peut ainsi parcourir une collection avec la boucle "each"

@each $key, $value in $grid {...}

Attention de bien mettre $key, $value car si vous ne mettez qu’une seule variable vous obtiendrez la concaténation de la clé et de la valeur...

$text-transform:(
text-uppercase: uppercase,
text-capitalize: capitalize,
text-lowercase: lowercase
);
@each $classname, $style in $text-tranform {
.#{$classname} {
text-transform: $style;
}
}

Les conditions

Les conditions en programation et en Sass fonctionent pareil. Il n'y a que la syntaxe qui change.

Pour écrire une condition en SASS nous utilisons l’instruction @if suivie de la condition à tester

@if $color == blue {
@include btn-blue;
}

Les fichiers partiels

L'idée des fichiers partiels est de diviser en plusieurs fichiers notre feuille de style.

Nous pourrons donc réutiliser certains fichiers d'un projet à un autre.

En Css la division de fichier est déconseillée car elle ralentie la page. Mais en Sass, il n'y a aucun problame car les fichiers seront compilés.

Création d'un fichier partiel

le nom du fichier doit etre précédé d'un underscore pour que Sass comprenne qu'il s'agit d'un fichier partiel.

par exemple, pour créer un fichier avec mes variables sass à l'interieur, je peux l'appeler "_variables.scss"

Importation

Lors de l'importation il ne faut pas :

  1. mettre l'underscore
  2. mettre l'extension .scss

Sass comprend qu'il doit aller chercher un fichier pariel meme sans ces informations

la syntaxe est la suivante: @import "nomdufichierpartiel";
